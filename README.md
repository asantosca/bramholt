# Bramholt

The single workspace for the whole company. Each folder is one function.

| Folder        | What lives here                                                        |
|---------------|------------------------------------------------------------------------|
| `company/`    | Strategy, roadmap, metrics, and the dated decision log.                |
| `product/`    | Product thinking — PRDs/specs, user research, and the backlog.         |
| `design/`     | Brand (logo, color, type, voice) and UX (flows, wireframes).           |
| `marketing/`  | Content, social, launch plans, and website copy.                       |
| `docs/`       | `public/` user-facing docs and `internal/` runbooks/SOPs.              |
| `dev/`        | The actual codebase(s). Placeholder until the stack is chosen.         |
| `ops/`        | Legal, finance, and the tool/subscription inventory.                   |

## Start here
1. `CLAUDE.md` — context Claude Code loads every session. **Fill in the TODOs first.**
2. `company/strategy.md` — mission, positioning, north-star metric.
3. `company/roadmap.md` — what you're doing now / next / later.

## Conventions
- Markdown everywhere except `dev/`.
- Dated files use `YYYY-MM-DD-title.md`.
- Decisions are logged in `company/decisions/`.
