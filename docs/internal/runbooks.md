# Internal runbooks (SOPs)

_How to run the business — the things you'd otherwise re-figure-out each time.
As a solo founder, write these down so the work is repeatable (and delegable later)._

## Planned runbooks
- [ ] Weekly review ritual (what to check, in what order)
- [ ] Publishing a blog post (draft → review → publish → promote)
- [ ] Shipping a release (when there's code)
- [ ] Responding to a new lead / signup

_Each runbook = a checklist. Keep them in this folder as `topic.md`._
